# SQL Queries

## eval_input_target_pairs.sql

Produces input-target pairs for evaluating the NMDC AI metadata suggestor.

**Input columns** (what the user provides during submission):
- `study_name` — always present
- `description` — study description, often abstract-quality text
- `notes` — free-form notes about the study

**Target columns** (what an LLM should predict from the input text):
- `env_broad_scale`, `env_local_scale`, `env_medium` — ENVO ontology terms, inferrable from study description
- `ecosystem`, `ecosystem_type`, `ecosystem_subtype`, `ecosystem_category`, `specific_ecosystem` — GOLD ecosystem classification hierarchy
- `analysis_type` — omics type (metagenomics, metabolomics, etc.)
- `sampleData` — interface/template choice (soil_data, water_data, etc.)

**Why these targets**: These fields describe the environmental and ecological context of the samples, which is typically described in the study text. An LLM reading "peatland soil microbial communities from Minnesota" should be able to predict env_broad_scale=forest biome, env_local_scale=peatland, env_medium=peat soil, etc.

**Not included as targets**: Fields like `collection_date`, `lat_lon`, `depth`, `samp_name` that require specific knowledge the submitter has but an LLM cannot infer from text alone. `geo_loc_name` is borderline — included because study descriptions often mention location.

**Source data**: Requires `local/nmdc_submissions.duckdb` generated by `export-submissions-to-duckdb`.

### Usage

```bash
# Query interactively
duckdb local/nmdc_submissions.duckdb < sql/eval_input_target_pairs.sql

# Export to TSV
duckdb -csv -separator '\t' local/nmdc_submissions.duckdb < sql/eval_input_target_pairs.sql > eval_input_target_pairs.tsv
```
